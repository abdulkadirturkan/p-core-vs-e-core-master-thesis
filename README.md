# P-Core / E-Core Performance Analysis with OpenMP Image Filtering

## VTune Microarchitecture Comparison on Intel Raptor Lake

This repository contains the implementation and benchmarking framework developed for my Master's Thesis research on heterogeneous CPU architectures.

The study investigates the performance characteristics of Intel Performance-Cores (P-Cores) and Efficient-Cores (E-Cores) using OpenMP-parallelized image filtering algorithms on Intel Raptor Lake processors.

The project includes:

- Parallel mean and Gaussian image filtering
- OpenMP thread affinity experiments
- P-Core vs E-Core performance comparison
- Intel VTune microarchitecture profiling
- Scalability, speedup, and efficiency analysis

---

## Thesis Topic

### P-CORE / E-CORE PERFORMANCE ANALYSIS:
### VTune Microarchitecture Comparison

Intel hybrid architectures combine high-performance P-Cores and energy-efficient E-Cores on the same processor die. This thesis evaluates whether E-Cores can provide comparable performance to P-Cores for OpenMP-based parallel image filtering workloads.

Experiments were performed on:

- Intel Core i7-14700KF (Raptor Lake-DT)
- Ubuntu 24.04 LTS
- OpenMP 5.2
- Intel VTune Profiler 2024
- OpenCV4

### Affinity Configurations
- P-Core only

- OS Scheduler (Unset)

Each configuration was executed 100 independent times.

## Key Findings

The experiments show that:

- P-Core and E-Core execution times are statistically nearly identical (<2% deviation)
- Workloads remain almost entirely cache-resident
- DRAM bandwidth utilization stays below 5%
- SIMD vectorization was not automatically generated by the compiler
- Both architectures execute mostly scalar floating-point operations
- OpenMP barrier synchronization and tile imbalance become dominant scalability factors at high thread counts

VTune analysis confirms that the workload is primarily compute-bound rather than memory-bound.

---

## Microarchitectural Observations

### VTune Results

The profiling workflow includes:

- HPC Performance Analysis
- Threading Analysis
- Memory Access Profiling
- Top-Down Microarchitecture Analysis (TMA)

Main observations:

- P-Core CPI: ~0.19–0.27
- E-Core CPI: ~0.28–0.33
- LLC misses: near zero
- Average memory latency: ~5 cycles
- Physical core utilization remained below 30%

Despite higher CPI values on E-Cores, end-to-end runtime remains comparable due to low effective core saturation.

## Parallelization Strategy

The implementation uses:

- OpenMP parallel regions
- Grid decomposition
- Tile-based workload distribution
- Zero-padding boundary handling

Image regions are partitioned into:

```math
⌈√p⌉ × ⌈p / √p⌉
```

tiles and assigned to OpenMP worker threads.

## Research Contributions

This work demonstrates that:

- Hybrid Intel architectures can achieve performance parity across heterogeneous core types for cache-resident image processing workloads.
- SIMD utilization is the dominant missing optimization.
- E-Cores provide competitive throughput with potentially superior energy efficiency.
